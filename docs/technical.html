Key Topics

Architecture

MedDialogue consists of several modular components:

  • TaskConfig: Define your medical task
  • ConversationConfig: Control conversation generation
  • DataPrep: Prepare training data
  • Trainer: Fine-tune models with LoRA
  • InferencePipeline: Run inference
  • SafetyChecker: PII, bias, validation

Data Preparation Pipeline

The pipeline transforms CSV rows into conversational training examples:

  1. Load and validate CSV data
  2. Generate question variations (grammatical + logical)
  3. Format responses (text, JSON, XML, Markdown)
  4. Create single-turn or multi-turn conversations
  5. Apply safety checks

Question Generation

16 question styles across 2 categories:

  • Grammatical (7 styles): Paraphrases of the same question
  • Logical (9 styles): Multi-field reasoning combinations

Safety & Compliance

  • PII Detection: HIPAA-compliant PHI detection
  • Bias Monitoring: Demographic distribution analysis
  • Clinical Validation: Medical terminology and code validation
