Traditional vs Conversational Fine-Tuning

‚ùå Traditional Instruction Fine-Tuning

Input: "Assess this patient: [clinical note]" Output: "Diagnosis: Malnutrition Present, Severity: Moderate"

Problem: Rigid patterns, fails on paraphrases

‚úÖ MedDialogue Conversational Fine-Tuning

Turn 1: "What's the malnutrition status?" ‚Üí [Full clinical assessment] Turn 2: "Why do you think that?" ‚Üí [Clinical reasoning with evidence] Turn 3: "How severe is it?" ‚Üí [Severity classification with justification]

Advantage: Natural, robust, handles variations

Quick Start Example

from meddialogue import MedDialogue, TaskConfig, ConversationConfig

# Define your medical task
task_config = TaskConfig(
    task_name="diabetes_screening",
    input_field="clinical_note",
    output_fields=["diagnosis", "hba1c_level", "risk_factors"]
)

# Configure conversation generation
conversation_config = ConversationConfig(
    single_turn_ratio=0.5,  # 50% single-turn, 50% multi-turn
    logical_style_ratio=0.4  # 40% logical reasoning questions
)

# Train model
trainer = MedDialogue(
    task_config=task_config,
    conversation_config=conversation_config,
    model_type="llama"
)
trainer.train_from_csv("diabetes_data.csv")

# Inference
response = trainer.infer(
    clinical_note="Patient presents with...",
    format="json"
)
print(response)
# {"diagnosis": "Type 2 Diabetes", "hba1c_level": "8.2%", ...}
